# 需求设计

1. 脚本、或是应用
2. 轻便，应用于服务器上
3. 目的：启动A服务器和B服务器。A服务器在中国大陆，受防火墙管控；B服务器在国外。A服务器供应商为腾讯云、华为云、京东云、阿里云等大厂，负责提供优质连接，通过开源VPN协议确保用户唯一并且全程连接极高安全加密，同时桥接B服务器获得AI服务以及跨境流量服务；B服务器主要目的有：1. 设置AI网关，例如cliproxyapi、axonhub、newapi等；2. 提供网络跨境服务，例如设置3x-ui面板等。
4. 该项目的应用的对象是没有能力或不清楚如何将国内外模型，使用在codex cli、opencode、claude code等gui、cli工具上大幅增加他们的工作效率的中小型公司、律所等。
5. 使用人数大概在30-100人左右
6. 目前我认为这套方案实现的关键在于这几点：
   1. 如何筛选优质开源VPN，能实现全程加密安全零信任网络需要
   2. 在A、B服务器中伪装3x-ui、VPN等工具：因为腾讯云、华为云等国内云服务商不允许出现代理的情况出现
   3. 设置白名单，即只能使用该服务器进行翻墙或者AI操作，拒绝流媒体使用

这是我对于整个项目的设想，我想知道：

1. 现有成熟解决方案是否存在？
2. 现有的解决方案还有哪些疏忽？
3. 还需要考虑哪些关键？
4. 如何实现？
